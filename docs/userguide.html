INTRODUCTION

This application allows the Customer Service Representatives (CSRs) of Bodgitt and Scarper, LLC., a broker of home improvement contractors, to generate a list of contractors that match a customer's criteria.

The application can be run in single-user (standalone) or multi-user (networked) mode. The multi-user mode uses a single "server" machine and one or more "client" machines.

AUDIENCE

These guidelines are provided for network administrators and users who are already familiar with the broad purpose of the project.

ASSUMPTIONS

These guidelines assume that a Java 2 Runtime Environment is already installed and all commands will be started from a Windows or UNIX/Linux shell command line.

SERVER OPERATION

Start the server from the directory where the database resides, using the following command line: 

java -jar runme.jar server

(If the runme.jar file is not located in the current directory, supply the full or relative path to the jar file.)

The server will normally produce output like the following: 

Apr 28, 2004 2:25:16 PM suncertify.protocol.server.SubcontractorCommandServer server
INFO: rmi://192.168.0.101:1099/SCJD-Subcontractor registered.

The server will run and handle requests until it is manually stopped. To stop the server, simply type <control-c>.

If another instance of the server is already running, the new server will produce error output like the following and then exit: 

Apr 28, 2004 2:55:58 PM suncertify.protocol.server.SubcontractorCommandServer server
SEVERE: Exiting -- java.rmi.AlreadyBoundException: rmi://192.168.0.101:1099/SCJD-Subcontractor

If a non-JRMP server is already listening at the standard (1099) port, the new server will product error output like the following and then exit: 

Apr 28, 2004 2:12:20 PM suncertify.protocol.server.SubcontractorCommandServer server
SEVERE: Exiting -- java.rmi.ConnectIOException: non-JRMP server at remote endpoint

If these (or other) errors occur, please consult your network administrator for help.

REMOTE CLIENT START-UP

Start the Graphical User Interface (GUI) client in "remote" mode using the following command line: 

java -jar runme.jar

(If the runme.jar file is not located in the current directory, supply the full or relative path to the jar file.)

  • If the GUI client has been connected to a remote server in the past and the server is running, the GUI table will automatically be filled with Subcontractors to choose from.
  • If this is the first time you have run the GUI in "remote" mode, the table will be empty.
  • If you have run the GUI in "remote" mode but the server is not running, you will see an error dialog window indicating that the server is not available. Simply press "OK". The table will be empty. You will now be able to select another server.

LOCAL CLIENT START-UP

Start the GUI client in "standalone" mode using the following command line: 

java -jar runme.jar alone

(If the runme.jar file is not located in the current directory, supply the full or relative path to the jar file.)

  • If the GUI client has been connected to a local database in the past, the GUI table will automatically be filled with Subcontractors to choose from.
  • If this is the first time you have run the GUI in "standalone" mode, the table will be empty.
GUI CLIENT OPERATION
Introduction

When the table contains Subcontractors and you have selected a Subcontractor, the window looks like this:

  • The "Search for Matching Subcontractors" button is enabled when the table contains Subcontractors and is disabled when the table is empty.

Searching for Matching Subcontractors

You can fill in the "Subcontractor Name" and/or "City" fields with the first few characters of the name of a subcontractor or city, then press the "Search for Matching Subcontractors" button, to narrow you search for subcontractors.

  • Uppercase and lowercase are ignored during searches.
  • Example: match string "rem" matches "Remodelers", "REM Builders" and "Remy, Inc." but not "Rebuilt".
Sorting the Table

You can click on any column heading in the table, to sort the entire table by that column. When you click the column heading more than once, the sort changes between ascending and descending order.
